QuickSearch decompilation index -- Minecraft 1.21.11 (Mojang mappings)
======================================================================

WHY THIS EXISTS
  mc_decompiled/src/ is gitignored and regenerated by `./gradlew genSources`.
  This index caches "just enough" verified facts so repeat tasks do not require
  re-reading the decompiled sources from scratch.

LAYOUT
  .index/<version>/<java/package/path>/<ClassName>.txt
  example:
    .index/1.21.11/net/minecraft/client/gui/screens/Screen.txt
  One directory per Java package; one text file per class that was inspected.

WHAT EACH CLASS FILE RECORDS (terse, only what was actually needed/verified)
  - Fully-qualified class name + the source path inspected (mc_decompiled/src/...)
  - Relevant method signatures, exact Mojang mapping (return type + params)
  - Relevant fields + their types
  - One-line behavior note when non-obvious
  - Date/context is optional; correctness against 1.21.11 is what matters

RULES
  - Add a line ONLY after verifying it in the decompiled source.
  - Keep it terse. No prose dumps, no full method bodies unless the algorithm
    itself is the point.
  - Plain text only. No markdown.
  - Never use Yarn names; never mix mapping namespaces.

For cross-cutting / non-class-specific notes, use mc_decompiled/.knowledge/.
